# Cadenza — Design Document

> 화면 설계, 온보딩, 브랜딩, 접근성 정책. PLANNING.md의 자매 문서.

---

## 1. 디자인 원칙

PLANNING 7장의 UX 원칙 재확인:

1. **러닝 중 한 손 조작** — 엄지로 닿는 영역에 핵심 컨트롤
2. **고대비** — 야외 햇빛 가독성 우선
3. **최소 정보 밀도** — 한 화면에 핵심만
4. **정직한 상태 표시** — 피치 변화/키 락 명확히 표시

여기에 추가:

5. **달리는 중에는 글자보다 숫자/아이콘** — 텍스트는 멈췄을 때 읽고, 달리는 중엔 색/모양으로 인지
6. **액션 결과는 즉시 시각/청각 피드백** — 토글 시 햅틱, BPM 변경 시 메트로놈 즉시 반영

## 2. 와이어프레임

ASCII 영역 분할 기반. 픽셀 단위가 아니라 **상대 비중과 배치**를 정의.

### 2.1 메인 플레이어 화면 (v0~v1 통합 와이어프레임)

> 이 와이어프레임은 최종 형태를 보여준다. 각 요소의 도입 버전은 아래 표 참조.

```
┌─────────────────────────────────┐
│                                 │  ← 상태바 (시스템)
├─────────────────────────────────┤
│  ◀  Cadenza             ⚙       │  ← 헤더 (좌: 뒤로가기 / 중: 타이틀 / 우: 설정)
│                                 │
├─────────────────────────────────┤
│                                 │
│      [곡 제목]                   │  ← 곡 정보
│      [아티스트 또는 파일명]        │
│                                 │
│      🎵 키 락 ON                 │  ← 상태 배지 (로컬: "키 락 ON" [v0a] / Apple Music: "⚠️ 피치 변경됨" [v1])
│                                 │
├─────────────────────────────────┤
│                                 │
│         ●●●●●●●●●●              │  ← 큰 BPM 디스플레이 영역
│                                 │
│           1 7 5                 │  ← 목표 BPM (가장 큰 숫자)
│            spm                  │
│                                 │
│         원곡 120 (자동) ✏️ → 175   │  ← 원곡 BPM (자동 감지, 탭하면 수정) → 목표 BPM
│           재생속도 1.46x          │  ← 계산된 비율
│                                 │
├─────────────────────────────────┤
│  ─────●────────────             │  ← 목표 BPM 슬라이더
│  140              200           │
│                                 │
│    [-1]  [160] [165] .. [180]  [+1]  │  ← 프리셋 + 미세 조절 [v0.5]
│                                 │
│  * [-1]/[+1]은 v0.5에서 추가. 러닝 중 땀 묻은 손으로
│    슬라이더 조작이 어려울 때 1 BPM 단위 조절용.
│    v0에서는 프리셋(5단위)만 제공 (가로 스크롤 가능)
│                                 │
├─────────────────────────────────┤
│                                 │
│  메트로놈   🔘 ON                 │  ← 메트로놈 토글
│  볼륨      ─────●──────         │  ← 메트로놈 볼륨
│                                 │
├─────────────────────────────────┤
│                                 │
│        ┌─────────────┐          │
│        │      ▶       │          │  ← 큰 재생 버튼 (엄지 도달 영역)
│        │              │          │
│        └─────────────┘          │
│                                 │
│      [📁 파일 선택]               │  ← 파일 변경 (재생 중엔 비활성)
│                                 │
└─────────────────────────────────┘
```

**영역 비중 (세로 기준)**
- 헤더: 8%
- 곡 정보: 12%
- BPM 디스플레이: 30% (가장 강조)
- BPM 컨트롤: 18%
- 메트로놈: 12%
- 재생 버튼: 20%

**핵심 원칙**
- 화면 중앙 30%에 큰 BPM 숫자 — 달리면서 흘긋 봐도 인지 가능
- 재생 버튼은 화면 하단(엄지 도달 영역)에 큼지막하게
- 슬라이더는 양손 잡고 멈춰서 조작, 프리셋은 한 손 가능

### 2.2.1 초기 상태 (empty state) 화면

앱 최초 실행 후 (온보딩 완료), 파일 미선택 상태.
메트로놈 단독 모드(2.2)와 유사하지만, 곡 선택을 유도하는 안내가 핵심.

```
┌─────────────────────────────────┐
│  Cadenza                    ⚙   │
├─────────────────────────────────┤
│                                 │
│         ♩                       │
│                                 │
│     음악을 선택하거나             │
│     메트로놈만 사용하세요         │
│                                 │
├─────────────────────────────────┤
│                                 │
│           1 7 0                 │  ← 기본 목표 BPM
│            spm                  │
│                                 │
├─────────────────────────────────┤
│  ─────●────────────             │
│  140              200           │
│  [160] [165] [170] [175] [180]  │
├─────────────────────────────────┤
│  메트로놈   🔘 OFF               │
│  볼륨      ─────●──────         │
├─────────────────────────────────┤
│        ┌─────────────┐          │
│        │   ▶ 메트로놈  │          │  ← 파일 미선택: 메트로놈 전용 재생
│        └─────────────┘          │
│                                 │
│      [📁 파일 선택]               │  ← 주요 CTA (강조 표시)
│                                 │
└─────────────────────────────────┘
```

**핵심 차이점 (메트로놈 단독 모드 2.2 대비)**:
- 재생 버튼 라벨: "▶ 메트로놈" (곡 없이 메트로놈만 재생됨을 명시)
- 파일 선택 버튼이 강조 표시 (Accent Primary 테두리)
- 안내 문구 "음악을 선택하거나 메트로놈만 사용하세요" 표시

### 2.2.2 에러 배너

에러 발생 시 화면 상단에 표시되는 배너. SPEC.md 2.7 "배너 노티 스타일" 참조.

```
┌─────────────────────────────────┐
│  ◀  Cadenza             ⚙       │
├─────────────────────────────────┤
│ ┌─────────────────────────────┐ │
│ │ ⚠️ 파일을 열 수 없습니다     │ │  ← 에러 배너
│ │    [다른 파일 선택]          │ │     배경: Background Secondary (#1A1A22)
│ └─────────────────────────────┘ │     테두리: Accent Warning (#FF8A3D) 1px
│                                 │     텍스트: Text Primary (#F5F5F7)
│      [곡 제목]                   │     액션 링크: Accent Warning (#FF8A3D)
│      ...                        │
```

**에러 배너 규칙**:
- 위치: 헤더 바로 아래, 곡 정보 위
- 자동 사라짐: 3초 후 페이드 아웃
- 수동 닫기: 좌/우 스와이프
- 높이: 고정 (2줄 텍스트 + 액션 버튼 1개)
- 복구 액션 버튼 필수 (SPEC.md 2.7 "확인 버튼만 있는 에러 금지" 준수)
- 연쇄 방지: 같은 에러 3회 반복 시 추가 표시 안 함

## 4. 브랜딩

### 4.1 컨셉 (Br1-c, Br2-a, Br3-a)

- **이름**: Cadenza
- **톤**: 절제된 우아함 + 리듬감. "음악의 자유 구간 위에서 달린다"
- **연관 이미지**: 동심원 파동, 리듬 패턴, 메트로놈 진자
- **피해야 할 것**: 너무 스포티한 운동 앱 느낌, DJ 컨트롤러 이미지

### 4.2 컬러 팔레트

**다크 베이스 + 비비드 액센트** (Br2-a 결정 반영)

| 역할 | 색상 | Hex | 사용처 |
|---|---|---|---|
| Background Primary | 거의 검정 | `#0A0A0F` | 앱 전체 배경 |
| Background Secondary | 다크 그레이 | `#1A1A22` | 카드, 섹션 구분 |
| Accent Primary | **시안 라임** | `#00E5C7` | BPM 숫자, 재생 버튼, 활성 상태 |
| Accent Warning | 따뜻한 오렌지 | `#FF8A3D` | "피치 변경" 배지, 경고 |
| Text Primary | 화이트 | `#F5F5F7` | 주요 텍스트 |
| Text Secondary | 라이트 그레이 | `#9A9AA5` | 보조 텍스트, 캡션 |
| Text Tertiary | 미디엄 그레이 | `#5A5A65` | 비활성 상태, 라벨 |
| Divider | 다크 그레이 | `#2A2A35` | 구분선 |

**액센트 컬러 선정 이유**: `#00E5C7` (시안-라임 중간) — 야외 가독성 높고, 다크 배경 위에서 BPM 숫자가 떠 보임. K-pop 시각 미와도 어울림. 너무 형광스럽지 않은 채도.

> 액센트 컬러는 시안 라임으로 시작하되, 실사용 후 마음에 안 들면 바꿈. 라이트 모드 대응은 v1.5+

### 4.3 타이포그래피 (runner-first redesign 2026-04-23 반영)

**SF Pro + IBM Plex Mono 조합**

| 역할 | 폰트 | 크기 | 용도 |
|---|---|---|---|
| BPM Display | SF Pro Rounded Bold | 56pt | 메인 플레이어 BPM 숫자 (이전 96pt에서 축소) |
| Track Title | SF Pro Display Heavy | 26pt | 플레이어 곡 제목 |
| Title 1 | SF Pro Display Bold | 28pt | 화면 제목 |
| Title 2 | SF Pro Display Semibold | 20pt | 섹션 제목 |
| Body | SF Pro Text Regular | 16pt | 본문, 아티스트명 |
| Caption | SF Pro Text Regular | 13pt | 보조 정보 |
| Mono Value | IBM Plex Mono Medium | 13pt | 상태 숫자 (BPM, ×1.00, ON/OFF) |
| Mono Timecode | IBM Plex Mono Regular | 12pt | 진행 시간 (1:52 / 3:04) |
| Mono Label | IBM Plex Mono Regular | 10pt, tracking 2px | 라벨 (TGT, KLK, SPM) |
| Mono Pill | IBM Plex Mono Medium | 11pt, tracking 1.5px | BPM pill 배지 (175 SPM) |

**핵심:**
- **BPM은 더 이상 96pt 주인공이 아니다** — 러너 사용 맥락에서 BPM은 한 번 설정하고 마는 상태값. 곡 제목·아트워크가 주인공.
- **모노스페이스는 상태 표현에만** — IBM Plex Mono는 숫자/상태값(BPM, 재생률, ON/OFF, 타임코드)에만. 장식 라벨에는 사용 안 함.
- IBM Plex Mono는 SIL OFL 라이선스 (상용 OK), Regular + Medium 두 웨이트만 번들 (~200KB).

**구현 노트**: IBM Plex Mono 번들 전까지는 시스템 monospaced (`Font.system(.., design: .monospaced)`)로 폴백.

### 4.4 앱 아이콘 컨셉 (Br1-c, 추상)

**메인 컨셉 후보 3가지** (실제 디자인은 별도 작업, 여기선 컨셉만 명세)

#### 후보 A: "리듬 동심원"
- 가운데에서 바깥으로 퍼지는 4개 동심원
- 각 원은 다른 두께/투명도 (강박-약박-약박-약박 시각화)
- 다크 배경 + 시안 라임 액센트
- 메트로놈/파동의 추상적 표현

#### 후보 B: "진자 슬래시"
- 메트로놈 진자의 잔상이 하나의 짧은 슬래시 형태
- 미니멀, 한 획
- 시안 라임 슬래시가 다크 배경 위에

#### 후보 C: "C 모노그램"
- "Cadenza"의 C를 음표 꼬리처럼 변형
- 가독성 우선, 폴더에서 알아보기 쉬움
- 가장 안전한 선택

**제 추천: A (리듬 동심원)** — Cadenza의 정체성과 가장 잘 맞음. 추상적이면서도 의미 있음.

> 실제 아이콘 제작은 v0 빌드 후 첫 러닝 검증 후로 미룸. 당분간 시스템 기본 아이콘 또는 임시 placeholder 사용.

### 4.5 다크 모드 / 라이트 모드

**v0~v1: 다크 모드 only**

근거:
- 러닝 환경(저녁/새벽/실내 어두움) 대부분이 다크가 유리
- 디자인 일관성 1세트만 검증
- 라이트 모드 대응은 v1.5+에서 사용자 요청 있을 때

## 5. 접근성 (a11y)

### 5.1 VoiceOver

- **모든 인터랙티브 요소에 라벨**: 슬라이더는 `accessibilityValue`로 현재 BPM 음성 안내
- **재생 버튼**: "재생" / "정지" (상태에 따라)
- **BPM 디스플레이**: "목표 BPM 175"로 음성 안내
- **상태 배지**: "키 락 켜짐" / "주의: 피치 변경됨"

### 5.2 Dynamic Type

**SwiftUI 기본 동작 활용**:
- BPM 큰 숫자는 고정 크기 (디자인 유지)
- 본문/캡션은 시스템 Dynamic Type 따름
- 최대 `xxxLarge`까지 지원, `accessibility*` 카테고리는 v1에서 검증

### 5.3 컬러 대비

WCAG AA 이상:
- Text Primary on Background Primary: `#F5F5F7` on `#0A0A0F` → 대비 18:1 ✅
- Accent on Background: `#00E5C7` on `#0A0A0F` → 대비 12:1 ✅
- Text Tertiary on Background: `#5A5A65` on `#0A0A0F` → 대비 4.6:1 ✅ (AA 4.5:1)

색만으로 정보 전달하지 않음 — 상태 배지는 색 + 텍스트 + 아이콘 조합.

### 5.4 모션 감소 (Reduce Motion)

`@Environment(\.accessibilityReduceMotion)` 활용:
- BPM 숫자 변경 시 부드러운 transition → 즉시 변경
- 메트로놈 시각화(파동 애니메이션) → 정적 표시

### 5.5 햅틱 피드백

러닝 글러브를 끼거나 화면을 못 볼 때 유용:
- 재생/정지 토글: 중간 강도 (`UIImpactFeedbackGenerator.medium`)
- 프리셋 변경: 가벼운 강도 (`light`)
- 메트로놈 on/off: 가벼운 강도

> 햅틱은 v0.5에 추가. v0는 시각/청각만.

### 5.6 화면 항상 켜기

설정에 `UIApplication.shared.isIdleTimerDisabled` 토글 제공. 기본 OFF (배터리 절약).

## 6. 언어 / 현지화 (L1-c)

### 6.1 정책

**한국어 + 영어 병행, 처음부터 i18n 셋업**

- 디폴트 언어: 시스템 언어 자동 감지
- 시스템 언어가 한국어면 한국어, 그 외 모두 영어
- 사용자가 설정에서 수동 변경 가능

### 6.2 구현 방식

**iOS 17+ String Catalog (`.xcstrings`) 사용**

- 기존 `Localizable.strings`보다 모던, Xcode 통합 우수
- SwiftUI에서 `Text("key")` 자동 해석

### 6.3 현지화 대상

| 카테고리 | 한국어 | 영어 |
|---|---|---|
| UI 라벨 | ✅ | ✅ |
| 온보딩 텍스트 | ✅ | ✅ |
| 에러 메시지 | ✅ | ✅ |
| 시스템 권한 다이얼로그 (Info.plist) | ✅ | ✅ |
| 앱 이름 | "Cadenza" (공통) | "Cadenza" |

> 앱 이름은 양쪽 다 "Cadenza" 유지 (브랜드 일관성). 한국어 부제는 필요 시 "케이던자" 음차 사용.

### 6.4 번역 정책

- v0~v1: JY 본인이 직접 번역
- 전문용어 (BPM, spm, 케이던스 등): 한국어 음차 + 영어 병기
  - 예: "케이던스 (cadence)", "spm (steps per minute)"
- 첫 등장 시 영문 병기, 두 번째부터는 한국어만

## 7. 인터랙션 디테일

> 모든 인터랙션의 기본 동작은 v0a부터 적용. 햅틱 피드백은 v0.5부터 (5.5장 참조).

### 7.1 BPM 슬라이더 동작 [v0a]

- 드래그 중: 실시간으로 BPM 숫자 변경 (재생 중이면 즉시 적용)
- 드래그 종료: 햅틱 피드백 (`light`) **[v0.5]**
- 슬라이더 양 끝(140, 200) 도달 시: 강한 햅틱 (`heavy`) **[v0.5]**

### 7.2 프리셋 버튼 [v0c]

- 탭: 즉시 해당 BPM으로 점프
- 현재 BPM이 프리셋과 가까우면(±2) 해당 버튼 강조 표시
- 햅틱 피드백 (`light`) **[v0.5]**

### 7.3 재생 버튼 [v0a]

- 탭 즉시 상태 전환 (latency 제로 인지)
- 정지 → 재생: 페이드 인 100ms (clicky 느낌 방지)
- 재생 → 정지: 즉시 멈춤 (사용자 의도 명확)

### 7.4 메트로놈 토글 [v0b]

- 토글 OFF: 메트로놈 즉시 정지, 음악은 계속
- 토글 ON (재생 중): 다음 비트부터 메트로놈 시작 (중간에 끼워넣기)
